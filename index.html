Noumenon builds a Datomic knowledge graph from your repository — immutable facts, temporal queries, and semantic annotations — designed for repositories too large for any context window, so you retrieve exactly what you need instead of hoping an LLM read the right files.

Why structured knowledge

As repositories grow, stuffing files into context windows becomes expensive and noisy. A queryable graph lets you retrieve exactly what you need.

Context windows don't scale

Dumping files into prompts degrades with repo size and dilutes signal with noise. A Datalog query returns exactly the entities a question needs, nothing more.

Answers you can't verify

When an LLM generates answers from raw context, you can't tell what it looked at or whether it missed something. Grounding answers in queryable, inspectable facts makes them auditable.

Code structure without history

Who changed this file, how often, and what else changed with it? Risk lives in the temporal dimension — churn, bus factor, co-change patterns — not just the current tree.

Three complementary layers

Each layer adds a different kind of information to the graph. The first two require no LLM at all. Use them independently or together.

●
No LLM required

Deterministic Facts

Git history, file structure, authorship, change frequency, and cross-file import graphs stored as immutable Datomic facts with stable entity identities. Fully reproducible.

●
LLM-powered

Semantic Annotations

Complexity ratings, safety concerns, architectural roles, function signatures, deprecation markers. Processed concurrently with up to 20 parallel workers, with automatic retry and checkpointing.

●
Iterative reasoning

Recursive LLM Querying

ask mode takes a natural-language question, formulates Datalog queries, reads results, reasons about gaps, and queries again — iterating until it has a grounded answer backed by data from the graph, not approximations from file contents.

Immutable

Datomic stores facts, not state. Every transaction is an accretion — nothing is overwritten. The full history of the graph is retained.

Composable

Datalog queries compose naturally. Join across git history, code structure, and semantics in a single query. No imperative glue code.

Measurable

Built-in A/B benchmarks. Compare graph-augmented answers against raw context on your own repo. Numbers, not claims.

Local

Runs on your machine. Your code stays yours. Only the LLM provider you choose sees file contents — and sensitive files are never sent.

Built for developers and agents

CLI and MCP tools for people who work in terminals and agents that query programmatically.

Concurrent & built for scale

Configurable parallelism (1–20 workers) with automatic retry on rate limits and transient errors. Incremental sync processes only new commits. Datomic's append-only storage and Clojure's persistent data structures keep memory predictable as the graph grows.

Git history as a first-class citizen

Commits, authorship, change frequency, co-changed files, and fix-commit ratios are all queryable entities in the graph — the temporal dimension of your codebase, not just the current tree.

MCP server

Expose the knowledge graph as MCP tools so Claude Desktop, Claude Code, or any MCP-compatible agent can query, import, and analyze repositories directly.

Multi-language import extraction

Deterministic cross-file import graphs for Clojure, Python, JS/TS, Go, Rust, Java, Elixir, Erlang, and C/C++. LLM analysis works with any language Git can track.

Cost transparency

Per-file token estimates before you start, per-file telemetry as it runs, and aggregate totals when it finishes. Query your own LLM spend with llm-cost-by-file and llm-cost-by-model.

Benchmark framework

40-question A/B test suite comparing knowledge-graph-augmented answers against raw context on your own repo. Deterministic scoring by default, LLM-judged scoring opt-in.

Sensitive file protection

Files matching known secret patterns (.env, *.pem, credentials) are recorded as entities but their contents are never read or sent to any provider.

Open data model

Datomic schema with stable entity identities. show-schema lists every attribute and type. Named queries are plain EDN files you can read, edit, and version control.

Perforce support

Works with Helix Core repositories via git-p4. Import a P4 depot as a Git mirror, then query it like any other repo. Helix4Git mirrors work directly.

Named queries

Pre-built Datalog queries in EDN format for common analysis tasks. Deterministic, reproducible, composable. Run from the CLI or through the MCP server.

  • Complexity and change hotspots
  • Bug-prone files by fix commit frequency
  • Co-changed file pairs
  • Top contributors per subsystem
  • Component dependency graph
  • LLM cost tracking by model and file

Or use ask mode to query the graph with natural language. The LLM formulates Datalog queries, reads the results, and iterates until it can answer from the data.

Get started

Requires JDK 21+ and Clojure CLI. Runs on your machine. See the README for standalone JAR and dependency options.

1

Clone and run

Or download the standalone JAR from GitHub Releases.

git clone https://github.com/leifericf/noumenon.git && cd noumenon
2

Import a repository

Point it at any local Git repo or a remote URL. Import is deterministic and takes seconds for most repos.

clj -M:run import /path/to/your/repo
3

Query or ask

Run a named query for instant results, or ask a question and let the LLM iterate over the graph.

clj -M:run ask -q "Which files have the most risk?" /path/to/your/repo
