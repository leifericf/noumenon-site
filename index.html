Precise, grounded answers
about your codebase.

Noumenon compiles your repository into a Datomic knowledge graph: git objects, code segments, files, and architectural components. AI agents query the graph instead of scanning raw source into context windows. Pre-compiled knowledge means fewer tokens, fewer iterations, and facts instead of hallucinations. Across 9 repos and 8 languages, graph-augmented answers scored 2.8× higher.

Three levels of understanding

The pipeline builds a multi-level graph. Queries traverse between levels naturally.

Micro

Code Segments

Functions, classes, and types with complexity ratings, code smells, safety concerns, purity analysis, and call graphs.

Mid

Files & Imports

Git history, authorship, change frequency, cross-file import edges, and per-file semantic summaries.

Macro

Components & Architecture

Logical subsystems identified from directory structure, import graphs, and file summaries. Component dependencies, layers, and categories.

How it works

Four stages compile raw git data into a queryable knowledge graph. Each stage builds on the last.

Import Git history & files deterministic
→
Enrich Import graph deterministic
→
Analyze Per-file semantics micro / LLM
→
Synthesize Components & architecture macro / LLM
→
Query / Ask / Serve Use the graph iterative
→
Introspect Self-improve autonomous

Import & Enrich

Deterministic foundation. Commits, files, authors, diffs, and resolved cross-file import edges. No LLM. Fully reproducible.

Analyze (micro)

Per-file understanding. An LLM reads each file and extracts code segments with complexity, smells, safety concerns, and architectural hints.

Synthesize (macro)

Codebase-level architecture. Queries the knowledge graph to identify logical components, classify files into layers, and map component dependencies.

Self-improving introspection

An autonomous loop that proposes changes, evaluates them against the benchmark, and keeps what works. Inspired by autoresearch.

Closed-loop optimization

Propose a change, re-evaluate, keep or revert. Five targets: prompts, examples, rules, source code, and model config. Evaluates across multiple repos to prevent overfitting.

Budget-controlled

Cap by iterations, hours, or dollar cost. Run --max-hours 8 for an overnight session that stops automatically.

On-device ML model

A pure-Clojure neural network learns which Datalog queries to try first. Trains locally at zero token cost.

Features

Git history as first-class data

Commits, authorship, change frequency, co-changed files, and fix-commit ratios. All queryable.

MCP server

The full knowledge graph as MCP tools for Claude Desktop, Claude Code, or any MCP client.

Multi-language imports

Deterministic import graphs for Clojure, Python, JS/TS, Rust, Java, C#, C/C++, Go, Elixir, and Erlang.

Cost transparency

Token estimates before you start, per-file telemetry as it runs, totals when done.

Sensitive file protection

.env, *.pem, credentials: tracked as entities but never read or sent to LLMs.

Architecture synthesis

Identifies components, layers, and dependencies from the full codebase. Language-agnostic.

Interactive TUI

Run noum with no arguments for a menu-driven terminal interface. Browse commands, select repositories and sessions from live data — no memorizing IDs.

Visual desktop UI

Electron app with a force-directed graph, three-level drill-down, floating Ask overlay, and Canvas rendering at 60fps. noum open

Concurrent and incremental

Up to 20 parallel workers with automatic retry. Incremental sync picks up new commits only.

Self-improving

Autonomous introspect loop proposes changes to prompts, rules, and code, then benchmarks them. Keeps what works, reverts what doesn't.

Share one knowledge graph across your team

Run Noumenon as a shared service for up to 200 concurrent users. One Docker command to deploy, token-based access, and all tools work transparently against the remote instance.

One-command deploy

Docker Compose with env-var config. No Clojure knowledge needed. Health checks, auto-restart, and named volumes included.

Role-based access

Admin tokens manage the server. Reader tokens for everyone else. The server manages its own git clones and refresh schedule.

Zero-config clients

noum connect <url> makes every CLI command and MCP tool route to the shared instance. Local paths are translated automatically.

$ docker compose up -d

Named queries

65+ pre-built Datalog queries in EDN, from file-level hotspots to component-level architecture. Deterministic, reproducible, composable.

  • Contributor and dependency graphs
  • LLM cost tracking by model and file

Get started

One command to install. See the README for details.

1

Install

Single binary. Auto-downloads everything it needs on first use.

curl -sSL https://noumenon.leifericf.com/install | bash

Or via Homebrew (macOS / Linux) or Scoop (Windows):

brew install leifericf/noumenon/noumenon scoop bucket add noumenon https://github.com/leifericf/scoop-noumenon && scoop install noumenon
2

Launch interactive mode

A menu-driven TUI that lets you browse commands by category, select repositories and sessions from live data — no memorizing IDs.

noum
3

Try the demo

Downloads a pre-built knowledge graph. No LLM credentials needed.

noum demo
4

Ask a question

Natural-language questions answered by iterative Datalog queries against the knowledge graph.

noum ask noumenon "Which files are the biggest risk hotspots?"
5

Open the visual UI

Force-directed graph visualization with three-level drill-down and a floating Ask overlay. Node.js must be installed.

noum open

Or import your own repo: noum import /path/to/your/repo
